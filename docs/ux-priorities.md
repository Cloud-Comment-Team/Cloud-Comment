# Переоценка UX-аудита и приоритеты CloudComment

Документ фиксирует состояние `main` на 13 июля 2026 года. Это планирование, а не список созданных GitHub issues. Оценки «гипотеза» требуют продуктовой или пользовательской проверки до реализации; оценки «факт» подтверждены текущим кодом.

Категории заданы пользовательским ТЗ: **A** — подтверждённый технический дефект; **B** — подтверждённый UX-блокер; **C** — privacy/security risk; **D** — продуктовый gap; **E** — улучшение качества; **F** — идея будущего развития. Размер: `S/M/L`. `COMMENTER` намеренно оценён как **E/P2** и не объявляется уязвимостью безопасности.

## Пакет P0

### Minimum

Minimum считается неделимым: выпускать отдельные части без остальных нельзя.

1. **Публичная identity:** убрать email из public response и DOM, ввести безопасный fallback имени; owner moderation и личный экспорт сохраняют законный доступ к email.
2. **Page identity:** единая клиентская и серверная функция удаляет fragment, нормализует origin/path и применяет утверждённую query-policy. Для интегратора остаётся явный canonical override.
3. **Widget draft и focus:** ввод редактора хранится в state независимо от полного `replaceChildren`; reaction, profile, sort и любой rerender не уничтожают текст/selection/focus, а POST 500 возвращает пользователя к сохранённому тексту.
4. **Сессия:** минимум — убрать чтение и удаление `ADMIN_AUTH_TOKEN_KEY` из widget, выдавать короткоживущий site-scoped commenter token и не позволять ему обращаться к глобальным account/admin API. Обычная SameSite cookie сама по себе не решает cross-site auth встроенного widget.

### Full

- Безопасно очистить или преобразовать исторические public author names, уже равные email.
- Подготовить aliases/merge для page identity, созданных tracking query, без потери комментариев и реакций.
- Добавить редактируемое публичное имя с историей согласия и безопасным fallback для старых аккаунтов.
- Перевести renderer на устойчивую модель состояния и точечные DOM updates без потери uncontrolled input; определить focus contract для reaction, sort, profile, auth и error rollback.
- Добавить диагностику canonical URL и session migration без записи токена/полного query в клиентские логи.
- Полная cross-site auth isolation: iframe/popup на CloudComment origin либо другой host-inaccessible session flow с ротацией, origin binding и отзывом. После совместимого окна удалить legacy bearer keys.

### Зависимости

- Продуктовое решение о формате публичного имени и допустимости совпадений.
- Перечень query-параметров: сохраняемые функциональные, удаляемые tracking и запрещённые sensitive.
- Additive migration страниц/алиасов и, при необходимости, публичной identity.
- Отдельные threat model и протоколы admin session и cross-site widget auth; SameSite/CORS/CSRF, popup/iframe messaging, site binding и отзыв рассматриваются вместе.
- Обновлённые privacy docs и data export/delete для новых identity/session данных.

### Definition of Done

- Anonymous public API и отрендеренный Shadow DOM не содержат email ни в одном состоянии.
- URL с fragment работает; tracking query не дробит тред; функциональный query следует документированной policy.
- Старые page records не теряют комментарии, replies, reactions и ownership.
- Введённый widget draft переживает reaction, открытие profile, смену sort и rerender; POST 500 сохраняет текст и возвращает focus/selection в редактор.
- Widget не читает и не удаляет admin key; host page не получает глобальный bearer; site-scoped token не работает на другом site и global account/admin API.
- Полная версия изолирует credential на CloudComment origin или эквивалентно недоступном host page контексте; login/logout/expiry/revocation и две вкладки работают.
- Backend unit/MVC/Testcontainers, widget unit/build и production-like Playwright проходят; privacy/security regression tests обязательны.
- Production smoke проверяет внешний origin, страницу с fragment/query, регистрацию, reload и отзыв сессии.

## Пакет P1

### Minimum

1. **Длинные обсуждения:** пагинация корней после первых 20 и собственные `PENDING` после reload.
2. **Безопасная модерация:** group undo для bulk operation и доступный Drawer с Escape, trap/inert и restore focus.
3. **Доступность:** включён axe `color-contrast` на всех обязательных темах и размерах.
4. **Обсуждения:** отдельный master-detail список тредов, а не redirect в очередь нарушений.
5. **Ответ владельца:** owner-scoped reply с безопасной публичной identity, realtime/notification и idempotency.
6. **Активация:** после создания открывается checklist установки, который сам подтверждает widget request и первый тестовый комментарий.

### Full

- Cursor pagination и восстановление позиции/сортировки для большого треда.
- В Discussions — поиск, site/page/status filters, permalink-ready URL и mobile list→detail.
- Bulk result показывает частичные ошибки, retry subset и доступную историю undo.
- Drawer заменяется постоянным detail там, где ширина позволяет; modal-поведение остаётся только на узком экране.
- Activation объясняет конкретный rejected origin, поддерживает backoff и прекращает polling после успеха/ухода со страницы.
- Contrast gate включает динамический widget accent и реальные состояния error/warning/disabled/focus.

### Зависимости

- UX-01/UX-20 до публичного owner reply.
- UX-02 до стабильных Discussions links и activation diagnostics.
- Существующие `replyCount`, replies endpoint, `operationId`, undo window и installation-status API.
- Согласованный master-detail и motion contract из `docs/design-system.md` и `docs/motion-system.md`.
- Test data на 45+ корней, pending от двух пользователей, mixed bulk result и rejected/healthy installation.

### Definition of Done

- 45 корней доступны без дублей и потери сортировки; replies загружаются независимо.
- Автор после reload видит свой pending; anonymous и другой пользователь — нет.
- Bulk undo восстанавливает всю допустимую группу и честно объясняет частичный конфликт.
- Drawer проходит Tab/Shift+Tab/Escape/restore focus; фон недоступен assistive technology.
- Axe contrast не отключён ни в одном обязательном UX spec; light/dark/custom accent зелёные.
- Владелец открывает тред из Discussions, публикует ответ, а внешняя вкладка получает его один раз.
- Новый сайт проходит create→install→widget seen→first test comment без ручного reload.
- Все сценарии проходят 390×844, 768×1024, 1440×900 и reduced-motion; production smoke не создаёт постоянный мусор.
